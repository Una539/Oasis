# 快速添加解析说明

本文档记录当前 Oasis 快速添加和输入标签相关的解析能力。实现主要位于：

- `src-tauri/src/quick_add.rs`：快速添加文本解析。
- `src-tauri/src/input_parser.rs`：`@` 标签输入分析和补全。
- `src-tauri/src/store.rs`：新增、编辑和状态归一化时应用解析结果。

## 解析结果

快速添加会把用户输入拆成以下结果：

| 字段 | 含义 |
| --- | --- |
| `content` | 去掉日期、截止词和优先级标记后的待办正文 |
| `planned_date` | 想做日期，格式为 `YYYY-MM-DD` |
| `due_date` | 截止日期，格式为 `YYYY-MM-DD` |
| `priority` | 优先级，范围为 `1` 到 `5`，默认值为 `3` |

日期只保留一种语义：如果解析出了 `due_date`，就不会同时保留 `planned_date`。也就是说，截止日期优先于想做日期。

## 日期解析

默认没有截止标记的日期会作为想做日期。

支持的中文日期包括：

- 特殊日期：`今天`、`今日`、`明天`、`明日`、`后天`、`後天`、`大后天`、`大後天`、`昨天`、`昨日`、`前天`、`前日`。
- 相对日期：`3天后`、`2周后`、`一个月后`、`1年前` 等。
- 星期表达：`周一`、`这周五`、`下周三`、`上周日`、`星期二`、`礼拜六` 等。
- 绝对日期：`7月10日`、`2026年7月10日`、`26年7月10号`。
- 数字日期：`2026-07-10`、`2026/7/10`、`7/10`。

支持的英文日期包括：

- 特殊日期：`today`、`tomorrow`、`tonight`、`yesterday`。
- 相对日期：`in 3 days`、`2 weeks later`、`1 month ago`。
- 星期表达：`monday`、`next fri`、`last sunday` 等。
- 月日表达：`Jul 10`、`July 10, 2026`、`10 July`。

中文和英文时间后缀会被一起移除，但当前保存结果仍然只保留日期，不保留具体时间。例如：

- `下周三下午3点 复诊` 会解析出日期，并把 `下午3点` 从正文里移除。
- `tomorrow at 9am call` 会解析出日期，并把 `at 9am` 从正文里移除。

没有年份的月日如果已经早于今天，会自动滚到下一年。例如在 `2026-07-05` 输入 `7/3 复盘`，日期会落到 `2027-07-03`。

## 想做日期与截止日期

以下标记会把后面的日期识别为截止日期：

- 中文：`截止`、`截至`、`到期`
- 英文：`deadline`、`ddl`、`due`、`by`

以下标记会把后面的日期识别为想做日期：

- 中文：`计划`、`安排`
- 英文：`scheduled`、`schedule`

没有标记时，日期默认是想做日期。

如果同一条输入里同时出现想做日期和截止日期，最终只保留截止日期。

## 优先级解析

默认优先级是 `3`。

支持以下写法：

- `!1` 到 `!5`
- `p1` 到 `p5`
- `P1` 到 `P5`
- `优先级 1` 到 `优先级 5`
- `優先級 1` 到 `優先級 5`
- `紧急`、`緊急`、`很紧急` 映射为 `5`
- `重要`、`很重要` 映射为 `4`

`!9`、`p0` 这类超出范围的标记不会被解析，会留在正文里。

## 标签输入解析

当前 `@` 标签输入用于交互式补全，不会在快速添加时自动把正文里的 `@标签` 创建成标签。

标签输入规则：

- `@` 必须出现在文本开头，或前一个字符是空白。
- `mail@work` 不会被识别为标签提及。
- 光标位于 `@query` 内时，会按标签名包含关系返回最多 6 个已有标签建议。
- 如果没有匹配项且 `query` 非空，会返回一个创建标签建议。
- 应用标签建议后，会把这段 `@query` 从输入文本中移除，并折叠多余空格。

## 示例

以下示例假设今天是 `2026-07-05`。

| 输入 | 正文 | 想做日期 | 截止日期 | 优先级 |
| --- | --- | --- | --- | --- |
| `买牛奶 明天` | `买牛奶` | `2026-07-06` | 无 | `3` |
| `交报告 截止周五 !2` | `交报告` | 无 | `2026-07-10` | `2` |
| `下周三下午3点 复诊 p5` | `复诊` | `2026-07-08` | 无 | `5` |
| `7月10日 还书` | `还书` | `2026-07-10` | 无 | `3` |
| `买牛奶 明天 截止周五 !4` | `买牛奶` | 无 | `2026-07-10` | `4` |
| `整理 !9` | `整理 !9` | 无 | 无 | `3` |

## 解析顺序和清理规则

1. 先解析优先级，并记录需要从正文移除的片段。
2. 再收集日期候选，跳过已经和优先级片段重叠的候选。
3. 日期候选按出现位置排序；重叠日期只保留第一个有效候选。
4. 根据日期前的标记判断是想做日期还是截止日期。
5. 移除日期、截止标记、想做标记、时间后缀和优先级标记。
6. 折叠多余空白，并去掉首尾分隔符。
7. 如果存在截止日期，清空想做日期。

## 已知边界

- 快速添加只保存日期，不保存具体时间。
- 多个非重叠日期会依次覆盖同类日期；如果最终存在截止日期，想做日期会被清空。
- `@标签` 的历史解析函数仍保留在代码里，但当前新增保存路径不会自动从正文创建标签。
