# Oasis Flow 开发路线图

> **版本**: 0.6.0 → 个人极简 Todo List
> **最后更新**: 2026-07-04

---

## 产品定位

Oasis Flow 是一款面向个人使用的极简 Todo List。它不试图成为项目管理工具，也不鼓励用户把生活拆成复杂系统；它的目标是让用户用最少动作完成三件事：

- 快速记下脑中的任务。
- 清楚看到今天真正要处理的事情。
- 安心保存、搜索、导出和同步自己的数据。

核心理念是：**默认极简，但完整功能可用**。默认界面应该适合大多数人直接使用；高级能力存在，但收起在更深一层，不干扰日常添加和完成任务。

## 明确不做

为保持方向稳定，Oasis Flow 不规划以下能力：

- 团队协作、成员分配、评论、审批。
- 项目管理、甘特图、看板、复杂层级任务。
- 番茄钟、习惯打卡、重度游戏化。
- 需要用户先配置大量规则才能开始使用的工作流。

## 竞品启发与取舍

- 学习 Todoist / TickTick 的快速添加：输入框应足够强，尽量少让用户离开键盘。
- 学习 Things 的日期模型：区分“今天想做”和“今天截止”，避免把计划意愿和真正期限混在一起。
- 学习 Microsoft To Do 的今日聚焦：打开应用后优先回答“今天我要做什么”。
- 避免 TickTick 式功能堆叠：搜索、重复、同步、导出都要有，但默认不把界面变成控制台。

## 默认信息架构

主导航收敛为三个入口：

| 入口 | 含义                                         | 默认行为             |
| ---- | -------------------------------------------- | -------------------- |
| 今日 | 今天想做、今天截止、过期但需要温和处理的任务 | 打开应用默认进入     |
| 以后 | 已安排到未来的任务和未来截止事项             | 不抢占今天注意力     |
| 灵感 | 没有日期、不确定何时处理的想法和任务         | 快速收集，低压力整理 |

高级功能不作为主导航出现。统计、已完成、导出、同步、设置、搜索筛选等能力进入次级入口或快捷入口。

## 近期路线

| 版本  | 主题             | 目标                                                                                                                                                                                                                                       |
| ----- | ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| v0.6  | 现有逻辑重塑     | 集中修改现有代码逻辑：移除可见 Tag 系统；旧数据中的 `tags` / `tag_ids` 隐藏保留；`@` 作为普通文本；保留并降级输入分词器；将主导航从 Today / Upcoming / Inbox / Outdated / Archived 重塑为 `今日 / 以后 / 灵感`；今日内区分“想做”和“截止”。 |

| v0.9  | 导出 / 导入      | 支持 JSON 完整导出与导入，保证用户数据可带走、可恢复。                                                                                                                                                                                     |

| v0.11 | 重复任务         | 支持简单重复任务，例如每天、每周、每月；完成后生成下一次任务，不引入复杂日历排程。                                                                                                                                                         |
| v1.0  | 本地优先同步基础 | 建立本地优先、可插拔的数据同步基础，优先支持用户指定文件夹同步，后续再扩展 WebDAV / Git / 云服务。                                                                                                                                         |

## 设计原则

- 快速添加优先于复杂整理。
- 今日视图只放会帮助用户行动的信息。
- 高级功能可以完整，但必须折叠。
- 数据属于用户，导出和备份是基础能力。
- 搜索是必需能力，不应依赖标签系统来找回任务。
- 完成任务后应有轻量正反馈，但不引入重度激励系统。
