# Oasis Flow 开发路线图

> **版本**: 0.8.0 → 个人极简 Todo List
> **最后更新**: 2026-07-11

---

## 产品定位

Oasis Flow 是一款面向个人使用的极简 Todo List。它不试图成为项目管理工具，也不鼓励用户把生活拆成复杂系统；它的目标是让用户用最少动作完成三件事：

- 快速记下脑中的任务。
- 清楚看到今天真正要处理的事情。
- 安心保存、搜索、导出和同步自己的数据。

核心理念是：**默认极简，但完整功能可用**。默认界面应该适合大多数人直接使用；高级能力存在，但收起在更深一层，不干扰日常添加和完成任务。

## 明确不做

为保持方向稳定，Oasis Flow 不规划以下能力：

- 团队协作、成员分配、评论、审批。
- 项目管理、甘特图、看板、复杂层级任务。
- 番茄钟、习惯打卡、重度游戏化。
- 需要用户先配置大量规则才能开始使用的工作流。

## 竞品启发与取舍

- 学习 Todoist / TickTick 的快速添加：输入框应足够强，尽量少让用户离开键盘。
- 学习 Things 的日期模型：区分“今天想做”和“今天截止”，避免把计划意愿和真正期限混在一起。
- 学习 Microsoft To Do 的今日聚焦：打开应用后优先回答“今天我要做什么”。
- 避免 TickTick 式功能堆叠：搜索、重复、同步、导出都要有，但默认不把界面变成控制台。

## 默认信息架构

主导航收敛为三个入口：

| 入口 | 含义                                         | 默认行为             |
| ---- | -------------------------------------------- | -------------------- |
| 今日 | 今天想做、今天截止、过期但需要温和处理的任务 | 打开应用默认进入     |
| 以后 | 已安排到未来的任务和未来截止事项             | 不抢占今天注意力     |
| 灵感 | 没有日期、不确定何时处理的想法和任务         | 快速收集，低压力整理 |

高级功能不作为主导航出现。统计、已完成、导出、同步、设置、搜索筛选等能力进入次级入口或快捷入口。

## 近期路线

| 版本  | 主题                 | 目标                                                                                                                                                                                                     |
| ----- | -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| v0.9  | SQL 存储与重复任务   | 将持久化后端从单个 `todos.json` 迁移到 SQL，建立可迁移、可扩展的数据结构，并为旧数据提供一次性安全迁移；优先完成简单重复任务，支持每天、每周、每月等常用规则，完成后生成下一次任务，不引入复杂日历排程。 |
| v0.10 | 导入、导出与备份     | 在 SQL 数据模型稳定后提供完整数据导出与导入；增加自动备份、手动备份和恢复入口，保证用户能够带走、迁移并恢复自己的数据。                                                                                  |
| v0.11 | 内置帮助与体验精修   | 在应用内提供面向首次使用者的使用帮助，覆盖三个列表、快速添加、日期语义、提醒、搜索及桌面端与移动端操作差异，并提供随时可再次打开的帮助入口；在移动端为完成任务、滑动操作等关键动作提供克制的马达震动反馈，同时统一并精修移动端与桌面端的视觉层级、动效、间距和交互状态。 |
| v0.12 | 核心重构与插件基础   | 重构核心模块，抽象存储、任务能力和扩展点接口，降低界面、业务逻辑与具体后端之间的耦合；将输入分词器的关键词与词表从解析流程中抽离，形成可注册、可组合的词表接口，为内置插件和未来第三方插件提供稳定基础。 |
| v0.13 | 英文本地化           | 建立统一的界面文案与本地化资源体系，提供完整、可切换的内置英文翻译；中文和英文作为核心应用正式支持的基础语言，并为翻译完整性和缺失文案提供检查机制。                                                    |
| v1.0  | 稳定核心与内置插件   | 稳定核心数据模型、抽象接口和插件生命周期；提供稳定的翻译扩展接口，使插件能够注册其他语言的文案、分词词表与本地化资源；保持默认安装轻量，把同步及其他可选、复杂能力以官方内置插件提供，并建立插件边界。       |

### 版本依赖

- v0.9 先完成 SQL 迁移，再基于稳定的数据模型实现重复任务，避免重复规则继续绑定旧的 JSON 快照结构。
- v0.10 在 SQL schema 和迁移机制稳定后再定义导入、导出与备份格式，避免短期内出现两套不兼容的数据格式。
- v0.11 聚焦用户可感知的完成度：内置帮助应能从应用内随时打开，不依赖用户访问项目仓库；帮助内容以首次使用流程为主，并与界面功能同步维护。该版本不与底层架构重构混在一起。
- v0.12 只建立插件所需的稳定边界，不提前堆叠插件功能；复杂可选能力从 v1.0 起以官方内置插件交付。
- v0.13 先用内置英文翻译验证本地化资源、文案回退和界面适配；v1.0 再将经过验证的能力开放为稳定翻译接口。
- 中文和英文由核心应用维护；其他语言在 v1.0 之后通过语言插件提供，不继续扩大核心安装包的翻译维护范围。
- 本地优先同步不再绑定核心版本实现，改为 v1.0 内置插件体系中的首批能力之一，后续可扩展用户指定文件夹、WebDAV、Git 或云服务。

## 设计原则

- 快速添加优先于复杂整理。
- 今日视图只放会帮助用户行动的信息。
- 高级功能可以完整，但必须折叠。
- 数据属于用户，导出和备份是基础能力。
- 搜索是必需能力，不应依赖标签系统来找回任务。
- 完成任务后应有轻量正反馈，但不引入重度激励系统。
- 首次使用不应依赖外部文档；核心操作需要在应用内有简短、可再次访问的帮助。
- 核心保持小而稳定；可选且复杂的能力优先通过插件扩展。
- 核心内置中文和英文；其他语言通过统一的翻译接口扩展。
